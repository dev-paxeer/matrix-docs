## Tabs

Tabs show content in switchable panels. Use them when the reader picks one path from several equal alternatives.

<Tabs>
  <Tab title="Node.js">
    ```javascript
    import { Matrix } from '@matrix/sdk';

    const client = new Matrix({ apiKey: process.env.MATRIX_API_KEY });
    const result = await client.query('What is the block height?');
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from matrix_sdk import Matrix

    client = Matrix(api_key=os.environ["MATRIX_API_KEY"])
    result = client.query("What is the block height?")
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl -X POST https://api.matrix.dev/query \
      -H "Authorization: Bearer $MATRIX_API_KEY" \
      -d '{"prompt": "What is the block height?"}'
    ```
  </Tab>
</Tabs>

```mdx
<Tabs>

    ```
  </Tab>
  <Tab title="Python">

    result = client.query("What is the block height?")
    ```
  </Tab>
</Tabs>
```

## Accordions

Accordion groups create collapsible sections for optional details, FAQs, or advanced content.

<AccordionGroup>
  <Accordion title="What is Matrix?">
    Matrix is an agent platform that gives AI agents persistent tools, memory, and chain access through MCP (Model Context Protocol) servers.
  </Accordion>
  <Accordion title="How do agents discover tools?">
    At boot, the executor loads the agent manifest, spawns each declared MCP server, and verifies that the server's advertised tools match the manifest exactly. Tools are then available to the planning layer via `matrix://` URIs.
  </Accordion>
  <Accordion title="Can I add custom tools?">
    Yes. Add an MCP server entry to your agent manifest, enumerate its tools, pin the package digest, and verify the bijection. See [Writing a Tool Bridge](/mcp/writing-a-tool-bridge).
  </Accordion>
</AccordionGroup>

```mdx
<AccordionGroup>
  <Accordion title="Question here">
    Answer here.
  </Accordion>
  <Accordion title="Another question">
    Another answer.
  </Accordion>
</AccordionGroup>
```

## When to use which

| Component | Use when |
| --- | --- |
| `Tabs` | The reader picks one of several equal alternatives (language, platform, OS). |
| `AccordionGroup` | Most readers will skip the content (FAQs, advanced options, troubleshooting). |
| `Steps` | The reader must do the work in a specific order. |

<Tip>
If readers do all the work, use `Steps`. If they pick one path, use `Tabs`. If most readers skip it, use `AccordionGroup`.
</Tip>
